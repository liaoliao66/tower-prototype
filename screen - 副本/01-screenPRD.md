# 铁塔检测运营指挥中心大屏 — 产品设计方案（PRD）

> 本文档与 `02-screen.html` 配套，适用于通信工程检测 / 铁塔运维指挥中心可视化建设验收与研发对齐。

---

## 1. 大屏设计定位与适用场景

### 1.1 定位

面向通信工程检测与铁塔运维场景的 **运营指挥中心级可视化大屏**：在单屏或拼接屏上集中呈现「订单—派单—现场作业—数据—报告—打印」全链路态势，支撑管理决策与一线调度。数据口径与现有 **PC 管理端、APP 现场端** 业务域对齐，便于验收与二次开发对接。

### 1.2 适用场景

| 场景 | 目标受众 | 核心价值 |
|------|----------|----------|
| 管理层汇报 | 分管领导、甲方汇报会议 | 按日/周/月展示合同履约、检测完成率、报告出具时效、异常闭环率；支持投屏 |
| 日常运营监控 | 运营中心值班长 | 准实时订单状态、在途派单、APP 在线与同步状态、设备在库概况 |
| 作业进度管控 | 项目经理、调度 | 按区域/项目/塔型聚合待办检测、现场采集进度、报告编制与打印审批队列 |

### 1.3 设计原则

- 信息密度可控、主次分明、**告警一眼可见**。
- 指标与原型 PC 侧栏（工作台、个人中心、合同部、营销、供应商、仪器、铁塔基础配置、基础数据、检测业务、运营看板、系统管理）及 APP（工单、检测、工作台、同步等）**同源可解释**，大屏不自造与业务系统冲突的口径。

## 2. 1920×1080 与 3840×1080 双分辨率适配布局架构

### 2.1 共同原则

- **12 列栅格 + 弹性区块**：顶部 KPI 带固定高度；中部主视觉与两侧模块按比例伸缩；底部为列表或次要指标带。
- **字号与间距按基准宽度缩放**：以 **1920px** 宽为基准；**3840px** 下主标题与 KPI 数字约 **1.25～1.4 倍** 线性放大，模块间距同比放大。
- **安全区**：四边预留 **24px（1920）/ 48px（3840）** 拼接缝与任务栏避让；告警角标与关键操作不贴边。

### 2.2 1920×1080（16:9）

- 推荐结构：**上约 12% KPI + 中约 58% 主区（地图或主趋势 + 订单/任务堆叠）+ 下约 22% 列表/趋势 + 下约 8% 状态条**。
- 单侧信息模块宽度不宜超过 **420px**；表格在 1080 高度下保持 **4～6 行** 可读行数。

### 2.3 3840×1080（32:9）

- 画布按 **「左：地图/区域 + 中：KPI 与趋势 + 右：列表与告警」** 三纵带划分，或 **「左半屏运营 + 右半屏作业」**。
- **禁止**仅横向拉伸单张地图：地图宽约 **40%**，其余为数据面板；列表可采用 **双列卡片** 或 **轮播**（同一数据集、同一口径）。

### 2.4 技术实现约束（与 HTML 一致）

- 使用 **CSS `clamp()`、`rem`、百分比高度** 与 **ECharts `grid` 百分比** 实现同一 DOM 双分辨率适配；必要时使用 `matchMedia` 切换列数。

## 3. 大屏模块与 APP / PC 端功能映射对照表

| 大屏模块 | 主要指标/内容 | PC 原型域 | APP 原型域 |
|----------|---------------|-----------|------------|
| 全局 KPI 总览 | 在检订单、今日完成单、报告待审、打印待批 | 合同部（订单/派单/跟踪/报告查阅）、检测业务（任务/报告/打印） | 工作台、工单列表状态聚合 |
| 订单与派单态势 | 状态分布、派单及时率、在途超时 | 订单管理、派单管理、派单跟踪 | 工单列表、工单详情 |
| 现场作业进度 | 采集完成度、同步积压 | 检测任务、检测数据录入 | 检测总览与各 `app_inspect_*`、同步页 |
| 报告与交付 | 编制中/待签发/已交付、平均周期 | 报告编制、报告查阅 | 间接来自任务完成与 PC 报告流 |
| 合同部履约 | 客户/工程维度订单量（占位扩展） | 客户管理、工程管理 | 工单与订单关联 |
| 资源与保障 | 设备在库率、供应商服务（占位） | 设备信息/出入库/台账、供应商名录 | 弱相关汇总 |
| 配置与质量基线 | 模板覆盖等（占位） | 任务/报告模板、标准规范 | 与现场表单一致性 |
| 安全与审计 | 登录异常、高危操作（占位） | 登录日志、操作日志 | 登录与同步失败率 |
| 运营扩展 | 高阶分析摘要 | 运营驾驶舱 / 数据看板 | 摘要上屏 |

## 4. 核心 KPI 与图表选型

| KPI | 含义 | 图表选型 |
|-----|------|----------|
| 在检订单数 | 未关闭检测订单 | 数字卡 + 日环比 |
| 今日现场完成 | 当日完成工单/任务单元 | 数字卡 + 近 7 日柱图 |
| 派单响应时效 | 派单至首次现场时长 | 折线 / 分位带 |
| 检测任务完成率 | 计划对比完成 | 环形图 / 进度条 |
| 超期未闭环 | 超 SLA 订单/任务 | 告警数字卡 + 列表 |
| 报告编制积压 | 各状态报告数 | 堆叠柱 / 漏斗 |
| 打印审批待办 | 待审数量 | 数字卡 + 横向 TOP 条 |
| APP 同步成功率 | 与同步模块一致 | 折线 + 仪表盘 |
| 区域订单分布 | 地理或区域维度 | 地图热力（有经纬度）/ 条形替代 |
| 塔型检测分布 | 与配置维度一致 | 玫瑰图 / 堆叠柱 |
| 设备在库率 | 在库/总数 | 仪表盘 |

**技术栈**：可视化实现采用 **ECharts 5**；系列色不超过 5 种，多类合并为「其他」。

## 5. 国央企深色科技风视觉规范

### 5.1 配色

- **背景**：主背景 `#0B1220`～`#0E1628`；卡片 `#111B2E`～`#152238`；分割线 `#1E2D45`。
- **主色**：科技蓝 `#165DFF`（主系列、选中、强调）。
- **语义色**：成功青绿、进行中琥珀、中性灰蓝。
- **告警**：一级 `#F53F3F`、二级 `#FF7D00`、三级弱化黄/琥珀；告警条使用半透明底。

### 5.2 字体

- 中文：思源黑体 / 阿里巴巴普惠体 / 微软雅黑；数字：DIN / Roboto（tabular）。
- **1920 基准**：大屏标题 28～32px，模块标题 18～20px，KPI 数字 36～48px，正文 14～16px，辅助 12～13px。

### 5.3 间距与圆角

- 模块间距 16～24px；内边距 16～20px；图表距卡片边缘 ≥16px。
- 圆角 4～8px；细边框 + 弱内高光，避免大块强渐变背景。

### 5.4 可读性

- 深色背景下表头与正文对比度满足抽检可读性；表格 hover 行高亮。

## 6. 交互逻辑

### 6.1 实时刷新

- KPI 与列表：**30～60s** 轮询或 WebSocket（实现期二选一）。
- 地图与趋势：**1～5min** 刷新，避免闪烁；刷新使用骨架或淡入。

### 6.2 区域下钻

- 地图：省 → 市 → 项目 → 站点（与 PC 筛选维度一致）。
- 无地图数据时：树图/矩形树/条形 **同级替代**。

### 6.3 模块联动

- 点选区域 → 过滤右侧订单列表、现场进度、报告积压（实现期接数据）。
- 点选塔型 → 联动检测分布与模板类指标（占位可配）。

### 6.4 异常告警弹窗

- 触发示例：超期突增、同步失败率超阈、登录失败风暴。
- **居中模态** + 摘要、影响范围、建议动作、跳转 PC 列表深链（可配置）。
- 同类告警 **5 分钟内去重合并**。
