# 铁塔检测运营指挥中心大屏 — 产品设计方案（PRD）

> 本文档与 `02-screen.html` 配套。**主受众为领导班子与甲方高层汇报**，弱化一线操作细节；**主视觉区中间必须为地图**，并在地图上标注当前检测任务覆盖的铁塔（站点）位置。

---

## 1. 定位与使用场景（领导视角）

### 1.1 产品定位

**领导驾驶舱级大屏**：一屏回答三件事——**全国/区域有多少塔在检、进展是否健康、风险在哪里**。不承载字段级、单据级的操作信息；明细由 PC / APP 承接，大屏仅作态势与决策提示。

### 1.2 典型受众与场景

| 场景 | 受众 | 关注点（示例） |
|------|------|----------------|
| 经营与履约汇报 | 公司领导、甲方分管领导 | 在检规模、阶段完成、是否按期交付报告 |
| 安全与风险总览 | 主管领导 | 超期、异常集中区域、需督办塔位 |
| 指挥调度开场 | 会议室投屏 | **地图上一眼看到塔在哪里、哪里标红** |

### 1.3 信息原则（刻意做「少而准」）

- **少**：首屏 KPI 控制在 **4～6 个**；不放长表格、不放多页签、不展开 PC 全菜单能力。
- **准**：指标口径与业务系统一致，仅保留领导听得懂的聚合名（如「在检铁塔座数」「本月完成检测」）。
- **一图胜千言**：**中间主区固定为地理地图**，塔点与状态颜色直接对应「在检 / 正常 / 需关注 / 风险」。

---

## 2. 布局架构（中间必须是地图）

### 2.1 总结构（1920×1080 与 3840×1080 通用）

自上而下四层，**第二层（主区）中间列仅布置地图**：

1. **顶栏**：系统名称、当前时间、极简操作（如告警演示 / 全屏）。
2. **KPI 窄带**：一行数字卡，仅核心结果指标。
3. **主区（视觉重心）**：**三列栅格 — 左摘要 | 中地图 | 右摘要**  
   - **中间列宽度占比约 50%～58%**（32:9 下仍保持中间为地图主体，左右为缩略图，禁止把地图挤成细条）。  
   - **中间列唯一主组件：中国（或可缩放省域）地图**，**在地图上以散点 / 动效点标注「当前处于检测流程中的铁塔（站点）」**；可选叠加 **风险塔** 不同符号或颜色。  
   - 左右列仅放 **单图趋势、环形占比** 等「一眼读完」的辅图，**不放与地图同维度的条形「区域排行」作为主视觉替代**。
4. **底栏（可选）**：一行文字级「今日需关注」摘要或 ≤3 条简讯，**不做宽表**。

### 2.2 双分辨率要点

- **1920×1080**：地图区高度建议不低于 **视口高度约 38%～45%**，保证塔点标注与省界可读。  
- **3840×1080**：中间地图列宽度 **≥ 总宽约 50%**；左右辅图可并排加宽，但**不削弱中间地图面积**。  
- 四边安全区、字号随宽度略放大等仍按原工程规范执行（见原 §2.1～2.3 原则，此处不重复展开技术参数）。

### 2.3 地图与「标塔」规范（强制）

| 项 | 要求 |
|----|------|
| 中间主区 | **必须为地理地图**（国家 / 省域二选一或下钻切换，实现期定数据源）。 |
| 塔位数据 | 来自 **站点/铁塔主数据中的经纬度**（与 PC「站点数据」「铁塔信息」、APP 工单定位一致）。 |
| 标注对象 | **当前检测流程未归档的铁塔（或站点级聚合）**；一点一站（或按领导口径聚合到工程级一点多塔需产品确认）。 |
| 状态表达 | 建议：**蓝色动效 = 检测中**，**橙色 = 临界**，**红色 = 风险/超期**；图例固定于地图卡片一角。 |
| 交互 | 支持缩放漫游；点击塔点弹出 **一行结论 + 跳转 PC 详情**（实现期配置 URL），大屏内不展开长表单。 |

---

## 3. 与 APP / PC 的关系（高层对照，不做细表）

领导大屏不逐条对照功能菜单。数据上仅保证：**塔点与检测任务 / 订单状态同源**（来自检测业务与合同履约主数据）；**经纬度与主数据一致**。PC 合同部、APP 现场作业仍为明细入口，大屏不做功能级映射表验收。

---

## 4. 核心指标（领导版短清单）

仅保留「汇报够用」的一层指标，**不展开二级拆解**：

| 指标（示例命名） | 领导含义 | 呈现方式 |
|------------------|----------|----------|
| 在检铁塔座数 | 当前有多少座塔在检测闭环中 | 顶栏 KPI 数字卡 |
| 本月检测完成 | 本月已闭环检测规模 | KPI 或左/右辅图趋势 |
| 报告按时率 / 在检订单 | 履约健康度（二选一或并列） | KPI |
| 需关注风险 | 标红塔位数量或等效风险指数 | KPI + **地图红点** |

其它如打印审批、设备在库等 **不默认上首屏**；若甲方要求可收进二级轮播或会议专版，本版 PRD 默认 **不展示**。

---

## 5. 视觉与交互（摘要）

- 视觉仍遵循国央企 **深色科技风**（背景深青灰、主色科技蓝、红橙告警），字体与对比度满足投屏距离阅读。  
- **刷新**：KPI 与塔点可 **1～3 分钟** 刷新；地图缩放状态尽量保持（实现期）。  
- **告警**：仅保留「影响面 + 建议动作」一句话弹窗，**不堆叠技术细节**。  
- **下钻**：地图省域下钻为可选项；领导汇报默认全国总览 + 标塔即可。
